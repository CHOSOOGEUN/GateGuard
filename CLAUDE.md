## 서비스 구조

### 사용자 앱 (React Native)
- 찍기 + 포인트 받기만
- iOS + Android 동시 배포
- 기능: 촬영 → AI 판별 결과 확인 → 포인트 수령 → 랭킹

### 관리자 웹 (React.js)
- 지자체 담당자용 PC 웹 대시보드
- 기능: 지도 시각화 · 분류 통계 · 검수 · 민원 접수

### 백엔드 (FastAPI)
- 사용자 앱 · 관리자 웹 공통 API 서버
- AI 모델 추론은 서버에서 처리 (엣지 아님)
- 비동기 처리: Celery + Redis

## 기술 스택

> 각 파트 담당자의 판단 하에 가장 효율적인 기술로 자유롭게 선택하여 진행.
> 아래는 현재 논의된 기준이며 파트별로 변경 가능.

```
AI / ML
- PyTorch
- YOLOv8 (Ultralytics) — 매체 탐지 (1단계, Bounding Box 반환 포함)
- EfficientNet-B0 — 2단계 불법 여부 분류 (M1 이후 보류)
- OpenCV — 전처리 + 얼굴 블러
- 학습 환경: Google Colab Pro

Backend
- Python 3.11 / FastAPI (담당자 재량으로 변경 가능)
- Celery + Redis (비동기 큐)
- PostgreSQL + ORM + 마이그레이션 툴 (담당자 선택)
- 클라우드 스토리지 (AWS S3 등) — 이미지 저장
- SMS 인증 (네이버 SENS 등)
- JWT 인증
- 이미지 크롭(Crop) 로직 — AI Bounding Box 기반, 백엔드에서 처리 후 스토리지 저장

Frontend
- React Native (사용자 앱)
- React.js + Tailwind CSS (관리자 웹)
- 네이버맵 API (지도)
- Chart.js (통계)
- 촬영 화면 가이드라인 UI (반투명 사각형 오버레이)

Infra
- Docker + Docker Compose
- AWS EC2 + Nginx + SSL (Let's Encrypt)
- GitHub Actions (CI/CD)
```

## AI 파이프라인

```
앱 촬영 (원본 이미지 전송, 가이드라인 UI만 표시)
    ↓
백엔드 수신
    ↓
YOLOv8 — 매체 탐지 (바운딩박스 + 분류값 반환)
    (현수막 / 전단지 / 스티커 / 명함 / 기타)
    ↓
백엔드 — Bounding Box 기반 이미지 크롭 후 스토리지 저장
    ↓
결과 반환 (매체 분류값 + 좌표)

--- M1 이후 추가 예정 ---
EfficientNet-B0 — 불법 여부 분류 (보류)
    (불법금융 / 불법도박 / 성인광고 / 기타불법 / 정상)
```

## 탐지 카테고리

### 1단계: 매체 분류 (YOLOv8) — M1 집중
1. 현수막
2. 전단지
3. 스티커 및 명함
4. 정상 광고

### 2단계: 불법 여부 분류 (EfficientNet) — M1 이후 보류
> 1단계 모델 안정화 및 시스템 완성 후 추가 진행
1. 불법금융
2. 불법도박
3. 성인광고
4. 기타불법
5. 정상광고

### 제외 사항
- 전화번호 추출 제외 (EasyOCR 미사용)
- 업체명 추출 제외

## 민원 처리 흐름

```
사용자 — 찍기 + 포인트 수령 (끝)

서버 / 관리자
    AI 판별 → 검수 → 민원 접수 → 처리 현황 추적
```

- 민원 자동 접수는 관리자 측에서 처리
- 사용자는 민원 과정에 개입 안 함
- 데이터셋만 넘겨줄 지 다시 확인
